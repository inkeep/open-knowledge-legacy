## Prerequisites

- **Bun >= 1.3.11** or **Node.js >= 22**
- **Git** installed and initialized in your project

## Initialize your project

<Steps>

<Step>
### Navigate to your project

```bash
cd your-project
```
</Step>

<Step>
### Run the init command

```bash
npx @inkeep/open-knowledge init
```

When run interactively, `init` prompts you to select which editors you use. Based on your selection it:

1. Scaffolds `.open-knowledge/` and writes `AGENTS.md` with the Open Knowledge agent instructions.
2. Registers the MCP server in each editor's config file (e.g. `.mcp.json` for Claude Code, `.cursor/mcp.json` for Cursor).
3. Writes the Open Knowledge section into the editor's agent instruction file — `CLAUDE.md` for Claude Code, `.cursorrules` for [[Cursor]], `.windsurfrules` for [[Windsurf]]. VS Code has no standard agent rules file.
4. Prints a content preview showing which files the watcher will track.

The `.mcp.json` entry written for Claude Code:

```json
{
  "mcpServers": {
    "open-knowledge": {
      "command": "npx",
      "args": ["@inkeep/open-knowledge", "mcp"]
    }
  }
}
```

The command is idempotent — safe to run again without side effects. Pass `--force` to overwrite existing entries (including the Open Knowledge section in `AGENTS.md` and per-editor instruction files), or `--no-mcp` to skip writing editor config files entirely.

**Upgrading an existing project:** if you ran `init` before, the `AGENTS.md` marker block is already present and `init` skips it by default. Run `open-knowledge init --force` to refresh the injected section with the latest agent instructions.
</Step>

</Steps>

## Start the editor

```bash
npx @inkeep/open-knowledge start
npx @inkeep/open-knowledge start --open  # also opens the editor in your browser
```

This starts the collaborative editor at `http://localhost:3000`. The port is configurable via `.open-knowledge/config.yml`. The `--open` flag launches your default browser automatically (macOS, Linux, and Windows).

<Callout type="info">
Starting the editor is optional. AI agents work without it -- the MCP server operates in disk-only mode when no editor is running.
</Callout>

## Create files from the sidebar

The file sidebar offers several ways to create a new markdown file or folder:

- **Header \`\` button** -- opens a dropdown with "New file" and "New folder". Defaults to the active file's parent directory, or project root when no file is open.
- **Right-click a folder row** -- "New file here" / "New folder here" targets that folder.
- `** (macOS) / **`\*\* (Linux/Windows)\*\* -- opens the new-file dialog with the active-directory default. The shortcut is ignored while focus is inside a text input, textarea, or content-editable element.
- **Empty workspace** -- a "Create your first file" button appears when no files exist yet.

New folders are always created with an initial file (default `index.md`) so they appear in the tree immediately. Filenames without a `.md` extension get one appended on submit.

## Connect AI agents

<Steps>

<Step>
### Open your project in an MCP-compatible editor

Claude Code, Cursor, Windsurf, or Codex all support MCP servers.
</Step>

<Step>
### Approve the MCP server

When prompted, approve the `open-knowledge` MCP server. The editor reads the `.mcp.json` entry created during init.
</Step>

<Step>
### Use the workflow tools

The agent now has access to three workflow tools:

- **`init-content`** -- Bootstrap articles from your existing codebase
- **`ingest`** -- Capture external sources as reference material
- **`research`** -- Gather sources and write structured findings
</Step>

</Steps>

## What got created

```
.open-knowledge/
  articles/          — Canonical knowledge articles
  external-sources/  — Ingested reference material
  research/          — Exploratory findings
  cache/             — Derived data (gitignored)
  catalogs/          — Auto-generated indexes (created when agents connect)
  AGENTS.md          — Navigation guide for AI agents
  config.yml         — Workspace configuration overrides
  .gitignore         — Excludes cache/
```

## Preview content scope

To see which files Open Knowledge will track -- without starting the server or changing anything -- run:

```bash
npx @inkeep/open-knowledge preview
```

This works before or after `init` and reflects the current state of your config. Useful after editing `content.include` or `content.exclude` in `.open-knowledge/config.yml`.
