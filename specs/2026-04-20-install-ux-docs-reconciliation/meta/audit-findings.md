## Is the spec implementable as-drafted?

**Yes, with two small-to-medium touch-ups.**

The two MED findings cluster around a single class of over-confidence:

1. **F01 + F12 + F20:** "verified in `mcp.test.ts:356`" is over-stated everywhere it appears. The test simulates the stderr string; it does not exercise the kernel ENOENT. The failure mode is mechanically correct by inspection of `editors.ts:23` + `init.ts:233`, but the test cited does not prove the failure occurs end-to-end. Fix is a phrasing weakening in §2.3 and §5 and in the audit report F5 (corrigendum).

2. **F02:** Mastra-writes-`npx`-in-MCP-configs citation chain is broken (cites a report section that doesn't contain the claim). External verification proves the claim IS factually correct, but the evidence trail is wrong. Fix is adding an external-source citation (the Mastra source file at GitHub) to the evidence file.

Neither of these MED findings blocks implementation — they're documentation-polish items that strengthen the spec's auditability. The actual design direction (D3-D8 LOCKED) stands up to scrutiny:

- D3 (bunx primary) matches shipped precedent + parent-spec alignment.
- D4 (Fumadocs `<Tabs groupId="bin">`) is architecturally viable and operationally supported by shipped dependencies.
- D5 (keep `npx` hardcoded + prereq + doc rationale + Option 2 escalation triggers) is a complete, reversible design.
- D6 (Fumadocs `<include>` + `scripts/sync-install-matrix.ts` + CI drift check) is architecturally verified in the evidence (Fumadocs source read + prior art from inject-markdown).
- D7 (prose `ok` / code blocks Tabs / lint on fenced-code) is coherent with D4.
- D8 (two-level prereq) follows cleanly from D5.

All 6 goals (G1-G6) have FR coverage. All §16 SCOPE entries are reachable via the D-rows. The EXCLUDE list is appropriately scoped (with a minor clarification needed per F09 on comment-level vs behavior-level edits).

**The LOW findings are drafting-polish items that improve auditability and internal consistency. None invalidate a decision; most are two-word or one-line fixes.**
