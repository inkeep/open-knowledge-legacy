## 2026-04-14 — Session 1: Test strategy designed; emulate evaluated and partially adopted

- Test strategy designed via `/tdd` methodology + existing harness review:
  - 5-layer pyramid: smoke (Bun compat) → unit (URL parser, token store, trust logic, gh-detect) → integration-real (CLI clone, subprocess relays, FORBIDDEN_UNTRUSTED, HEAD-drift) → integration-mocked (Device Flow state machine, PAT scope validation) → E2E (Playwright).
  - ~95% of spec is autonomously verifiable in CI. Unverifiable carve-out: (1) real Device Flow end-to-end against github.com (requires human approval — one-time pre-ship manual smoke), (2) real OS keychain on CI without keychains (fallback file store is autonomous).
  - Key utilities to build: `createBareRepo()` for `file://` clone targets, mock GitHub HTTP server for Device Flow, `mockGhBinary()` for Tier A, `simulateServerRestart()` for HEAD-drift.
  - TDD tracer bullet: `open-knowledge clone file:///tmp/bare.git /tmp/target` → server starts → `/api/documents` returns cloned files. Exercises URL parser + simple-git + auto-init + server-lock + documents API. Builds breadth from this spine via vertical slicing (one red→green cycle per behavior; never batch).
- Evaluated [vercel-labs/emulate](https://github.com/vercel-labs/emulate): production-fidelity GitHub API emulator (+ 6 other services), stateful, Hono-based.
  - **Adopts now (narrow):** Replaces our mock server for REST endpoints `/user`, `/user/repos`, `/repos/{o}/{r}`. Cleaner, more realistic, future-proof for push/fork/MCP-GitHub features.
  - **Does NOT cover:** Device Flow endpoints (`/login/device/code`, `device_code` grant type) — we keep a small local Device Flow mock.
  - **Does NOT cover:** git clone protocol (git-upload-pack) — our `file://` local bare repo pattern remains correct.
  - Added Future Work Identified: contribute Device Flow support to emulate upstream (~150 LOC PR). Turns emulate into single-emulator solution + ecosystem win.
