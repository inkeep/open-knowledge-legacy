# Evidence: Editor integration surfaces for clone feature

**Date:** 2026-04-14
**Sources:** `packages/app/src/components/FileTree.tsx`, `EditorPane.tsx`, `EditorHeader.tsx`, `packages/server/src/api-extension.ts`, `packages/cli/src/content/init.ts`, `packages/cli/src/config/loader.ts`

## Q1: Empty-state detection

Empty state = `documents.length === 0` from `/api/documents` polling (FileTree.tsx:394-416). Reactive via 5-second interval (FileTree.tsx:275-300). No concept of "no project loaded" — always assumes a server is running. ProjectPicker component keys on `documents.length === 0 && !activeDocName && !loading`.

## Q2: Subprocess spawning from editor

`POST /api/save-version` (api-extension.ts:1045-1113) is precedent for "endpoint that does git work." Pattern: HTTP endpoint → server-side git operations → JSON response.

For clone: `POST /api/local-op/clone` spawns `open-knowledge clone --json <url>` as child process, streams JSONL via chunked `Transfer-Encoding`. After clone, the subprocess starts a new server instance for the cloned dir, reads port from `server.lock`, returns it in the final JSONL line. Browser redirects.

No existing subprocess-spawn or SSE pattern in the codebase, but Node `ServerResponse` supports `res.write()` + chunked encoding natively. saveVersion is the closest pattern.

## Q3: Config scaffold detection

`CONFIG_YML_CONTENT` in `content/init.ts:91-124` starts with `# Open Knowledge — workspace configuration`. No sentinel marker. Config loader (`loader.ts:1-95`) reads YAML and merges with defaults — does not distinguish scaffolded vs external.

Recommended: add sentinel first line `# Generated by Open Knowledge init` to the template. Trust check: `raw.startsWith('# Generated by Open Knowledge init')` → if false, config is external → trigger trust prompt.

## Q4: Trust-pending vs preview mode

EditorMode is `'wysiwyg' | 'source' | 'diff'` (EditorPane.tsx:11-16). Diff mode is per-document, triggered by clicking a timeline entry (EditorPane.tsx:38-51). Sets `previewEntry` to a specific historical version of the active doc.

Trust-pending is project-level, not document-level. Should NOT reuse diff mode. Instead:
- Add `trustPending: boolean` to EditorPane
- Render a project-level banner with [Review config] [Trust and enable] [Keep read-only]
- Disable all editor controls when trustPending
- Force `editorMode = 'source'` (read-only markdown view)
- On trust confirmation: set `trustPending = false`, restore normal editing
- Diff mode remains the document-level timeline tool

Header conditional (`EditorHeader.tsx:71-94`) keys on `!isDiffMode` for showing the Visual/Markdown toggle. For trust, add `&& !trustPending` to that condition.
