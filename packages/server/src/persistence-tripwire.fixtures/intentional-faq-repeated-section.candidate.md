# Frequently Asked Questions

This document collects answers to common questions. Some Q&A intentionally repeats verbatim across sections so that each top-level audience finds a self-contained answer in one place.

## For new contributors

### How do I file a bug?

Open an issue on the tracker. Include reproduction steps, expected behavior, and observed behavior.

### How do I propose a feature?

Open a discussion thread first. We use discussions to scope the change before any code lands.

## For maintainers

### How do I file a bug?

Open an issue on the tracker. Include reproduction steps, expected behavior, and observed behavior.

### How do I triage incoming issues?

Apply the `needs-triage` label, ask for missing reproduction info, and assign a milestone within one business day.

## For external integrators

### How do I file a bug?

Open an issue on the tracker. Include reproduction steps, expected behavior, and observed behavior.

### How do I report a security vulnerability?

Email the security alias rather than filing a public issue.
