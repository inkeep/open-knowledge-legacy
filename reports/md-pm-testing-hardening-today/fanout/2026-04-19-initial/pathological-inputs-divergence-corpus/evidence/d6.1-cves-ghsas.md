## 2. Per-CVE Detail Findings

### marked

**CVE-2021-21306 / GHSA-4r62-v4vq-hr96 — Underscore ReDoS** (CONFIRMED)
- Affected: `>=1.1.1, <2.0.0`. Patched 2.0.0. CWE-400. CVSS 5.3. Published Feb 8 2021.
- No PoC disclosed in advisory.
- URL: https://github.com/markedjs/marked/security/advisories/GHSA-4r62-v4vq-hr96

**CVE-2022-21680 / GHSA-rrrm-qjm4-v8hf — Cubic catastrophic backtracking in `block.def`** (CONFIRMED)
- Affected: `<4.0.10`. Patched 4.0.10. CWE-1333 + CWE-400. CVSS 7.5. Published Jan 14 2022.
- PoC payload (verbatim from advisory):
  ```js
  marked.parse(`[x]:${' '.repeat(1500)}x ${' '.repeat(1500)} x`)
  ```
- Cubic-time backtracking in the block-level link reference definition regex.
- Reporter: @makenowjust (`recheck` author).
- URL: https://github.com/markedjs/marked/security/advisories/GHSA-rrrm-qjm4-v8hf

**CVE-2022-21681 / GHSA-5v2h-r2cx-5xgj — Exponential ReDoS in `inline.reflinkSearch`** (CONFIRMED)
- Affected: `<4.0.10`. Patched 4.0.10. CWE-1333. CVSS 7.5. Published Jan 14 2022.
- PoC pattern: nested unclosed bracket sequences `[x]: x \n [](\[](\[](...)`. Exponential blow-up in reference-link search.
- URL: https://github.com/markedjs/marked/security/advisories/GHSA-5v2h-r2cx-5xgj

**CVE-2018-25110 / GHSA-p9wx-2529-fp83** (CONFIRMED)
- Affected: `<0.3.17`. Patched 0.3.17. Originally 2018, republished as GHSA May 2025.
- PoC at [Checkmarx Vulnerabilities-Proofs-of-Concept repo](https://github.com/Checkmarx/Vulnerabilities-Proofs-of-Concept/tree/main/2018/CVE-2018-25110).

### markdown-it

**CVE-2022-21670 / GHSA-6vfc-qv3f-vr6c — Newline rule ReDoS** (CONFIRMED)
- Affected: `<=12.3.1`. Patched 12.3.2. Moderate. Published Jan 8 2022.
- PoC payload (verbatim from advisory):
  ```js
  md.render(`x ${' '.repeat(150000)} x  \nx`)
  ```
- Vulnerable regex: `/\s+$/` at line 23 of `lib/rules_inline/newline.js` — used to strip trailing whitespace, but the `$` anchor with greedy `\s+` produces quadratic time on long whitespace runs. Fix in commit `ffc49ab`.
- URL: https://github.com/markdown-it/markdown-it/security/advisories/GHSA-6vfc-qv3f-vr6c

**CVE-2025-7969 / GHSA-vgm8-mvfh-rj89 — XSS in renderer** (CONFIRMED)
- Affected: `14.1.0`. CWE-79. CVSS 6.9 (v4 vector). Published Aug 21 2025.
- Located in `lib/renderer.mjs`. Filed by Fluid Attacks.
- URL: https://github.com/advisories/GHSA-vgm8-mvfh-rj89

**CVE-2026-2327 / GHSA-38c4-r59v-3vqw — Linkify ReDoS** (CONFIRMED)

## 3. Bug Class Taxonomy

### CWE-1333 / CWE-400 — ReDoS / catastrophic backtracking (DOMINANT class — 9 of 20)
- marked CVE-2021-21306, CVE-2022-21680, CVE-2022-21681, CVE-2018-25110
- markdown-it CVE-2022-21670, CVE-2026-2327
- markdown-it-toc-and-anchor (infinite loop, control flow rather than regex)
- @ckeditor/ckeditor5-markdown-gfm CVE-2021-21254
- markdown-link-extractor CVE-2021-43308
- markdown (npm) GHSA-wx77-rp39-c6vg

**Pattern:** all hot regexes touching link-reference, autolink/linkify, newline, or em-marker grammars are recurring offenders. Cubic and exponential blowups appear with crafted inputs around 1.5K–150K chars. Common mitigation in advisories: "run on a worker thread with a time limit."

### CWE-79 — XSS via parser/renderer output
- remark-html CVE-2021-39199 (unsafe defaults — documentation/security-posture bug, **CVSS 10.0**)
- markdown-it CVE-2025-7969 (renderer bug)
- markdown-it-decorate CVE-2020-28459 (`javascript:` URLs / event handlers — unpatched)
- markdown-pdf CVE-2023-0835 (server-side XSS in headless-Chrome rendering pipeline → local file read)
- markdown-to-jsx CVE-2024-21535 (`src` attribute not sanitized like `href`)
- mdast-util-to-hast CVE-2025-66400 (class-attribute injection via character references)

### CWE-1321 — Prototype pollution (DOMPurify ecosystem)
- DOMPurify CVE-2024-48910, CVE-2024-45801

### Mutation XSS (mXSS) — re-contextualization, browser parser quirks
- DOMPurify GHSA-h8r8-wccr-v5f2 (`<xmp>`/`<noscript>`/etc. re-parse)
- DOMPurify CVE-2025-26791 (SAFE_FOR_TEMPLATES regex)

### CWE-674 — stack exhaustion
**No advisories found in 2020+ scope.** Markdown parsers do recurse on nested blockquotes/lists, but no published CVE/GHSA assigned an explicit stack-overflow class in the JS ecosystem in this period. Crashes are documented in issue trackers (see D6.2) but never escalated to advisory.

## 5. Cross-Cutting Findings

1. **The "abandoned plugin" risk is real.** `markdown-it-decorate` (CVE-2020-28459), npm `markdown` (GHSA-wx77-rp39-c6vg), and `markdown-pdf` (CVE-2023-0835) all have advisories with **no available patch** — fixed only by migrating away from the package.

2. **ReDoS is by far the most common CWE class** — ~9 of 20 advisories. The fixes consistently move from greedy-with-backtracking patterns to bounded or linear formulations (e.g., the markdown-it newline-rule fix in commit `ffc49ab`).

3. **Default safety is the second-largest class.** remark-html CVE-2021-39199 (CVSS 10.0) reflects a broader failure of "secure defaults" in the unified pipeline before v13.

4. **DOMPurify is a load-bearing dependency for safe markdown rendering** — but accumulated four 2024–2026 CVEs/GHSAs of its own (two prototype-pollution, two mXSS). Markdown apps that route through DOMPurify inherit those windows.

5. **Plugin ecosystems amplify the surface** — markdown-it has 1 core CVE since 2020 but plugins (`-decorate`, `-toc-and-anchor`, ckeditor-gfm, `markdown-link-extractor`) collectively contribute several more.

6. **No micromark/remark-core CVE in 2020+** — the architectural choice of a state-machine character-by-character tokenizer (vs regex-driven parsing) appears to have eliminated the ReDoS *class* by construction, at least so far.

## 6. Gaps / Follow-ups

- No direct CVE for `gray-matter` (frontmatter), `vfile-message`, or `vfile-statistics` was located. These are utility/diagnostic packages without parser surfaces, but are routinely chained into pipelines.
- `markdown-it-sub`, `markdown-it-sup`, `markdown-it-footnote`, `markdown-it-container` — no GHSA filed (absence of evidence, not evidence of absence).
- The published markdown-it `docs/security.md` URL referenced in older docs returns 404; current security guidance is folded into the README and security tab.
