# Chroma Product Capabilities Evidence

## Self-Description
Chroma describes itself as "the open-source data infrastructure for AI" with built-in retrieval capabilities. The official docs make no mention of knowledge management, editing, authoring, or human-readable content features.

## Core API
The core API is "only 4 functions" (from README):
1. Create/manage collections
2. Add documents (with automatic tokenization and embedding)
3. Query by similarity
4. Filter by metadata

## Data Model
- **Collections**: Analogous to tables in relational databases. Store embedded documents + metadata.
- **Documents**: Raw text content stored alongside embeddings.
- **Embeddings**: Vector representations generated by configurable embedding models.
- **Metadata**: Key-value pairs (string keys; string/int/float/bool values) attached to documents.
- **IDs**: Each record requires a unique string ID.

## Search Capabilities
- Dense vector search (HNSW-based)
- Sparse vector search (BM25 and SPLADE, added November 2025)
- Full-text search
- Regex search (added June 2025)
- Metadata filtering at query time
- Hybrid search (combining dense + sparse)
- Multi-modal retrieval (text, images, audio via OpenCLIP)

## Product Lines (as of 2026)
Chroma's website now lists three product lines:
1. **Database** - Core ChromaDB vector storage and retrieval
2. **Sync** - Data synchronization from GitHub, S3, Web
3. **Agent** - AI agent integration (Package Search MCP, Context-1 model)

## Context-1 Research Model
A 20B parameter language model purpose-built for multi-turn retrieval:
- Functions as a specialized search agent that produces ranked document lists
- "Self-editing" capability: actively discards tangential documents mid-search via `prune_chunks` tool
- Achieves "retrieval performance comparable to frontier-scale LLMs at a fraction of the cost and up to 10x faster inference speed"
- Returns documents to downstream reasoning models rather than generating answers

## Storage Architecture
Three-tier storage:
1. Incoming embeddings stored in brute force buffer
2. Transferred to HNSW-based vector cache
3. Persisted to disk using Apache Arrow format

Cloud architecture:
- Query layer with fast memory cache
- SSD cache layer
- Storage layer using S3/GCS for vectors, metadata, and indexes

## Deployment Options
- **In-memory** (ephemeral, for development)
- **Persistent** (file-based, SQLite-backed local storage)
- **Self-hosted server** (HTTP client to self-managed Chroma instance)
- **Chroma Cloud** (managed serverless)

## Rust Rewrite (2025)
Core rewritten in Rust, eliminating Python GIL bottlenecks:
- True multithreading
- Up to 4x performance improvement for writes and queries
- Languages breakdown: Rust 66.4%, Python 17.2%, TypeScript 7.6%

## What Chroma Does NOT Have
- No editing/authoring interface
- No human-readable content layer
- No markdown or text editing
- No WYSIWYG or rich text editor
- No version history for content (only collection forking for embeddings)
- No collaboration features beyond multi-tenancy isolation
- No knowledge graph or structured knowledge representation
- Purely programmatic access (SDKs, API, MCP)
