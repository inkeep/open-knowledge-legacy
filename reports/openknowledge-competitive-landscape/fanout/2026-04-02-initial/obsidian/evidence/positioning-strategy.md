# D6: Positioning & Strategic Direction - Evidence

## Positioning

**Tagline:** "A second brain, for you, forever."

**Five principles (from obsidian.md/about):**
1. **Yours** — Free access, everyone should have tools to think clearly
2. **Durable** — Open file formats, no lock-in
3. **Private** — Data on your device, E2E encryption for cloud services
4. **Malleable** — Highly customizable and extensible
5. **Independent** — "100% supported by our users, not investors"

**Positioning analysis:** Obsidian positions at the intersection of:
- **Local-first** (vs. cloud-first like Notion)
- **Markdown-native** (vs. proprietary formats like Notion blocks)
- **Customizable** (vs. opinionated like Apple Notes)
- **Single-player** (vs. collaborative like Google Docs)
- **Community-driven** (vs. top-down feature development)

## Founding History

- **Co-founders:** Shida Li (CTO) and Erica Xu (COO). Both University of Waterloo alumni.
- **Prior product:** Dynalist (outliner tool), built together before Obsidian.
- **Obsidian developed March 2020** during COVID quarantine. First public release October 2022.
- **Shida Li's background:** Software engineering at LinkedIn (created Dr. Elephant, a Hadoop performance tool) and Dropbox. Infrastructure-focused.
- **CEO transition:** Steph Ango (kepano) joined as CEO in February 2023. Previously a community power user and theme designer (Minimal Theme). Was head of design at a Y Combinator startup.

## Steph Ango's Philosophical Stance

Ango is unusually opinionated for a CEO, publishing essays on his personal blog (stephango.com):

**"File over app"** — founding philosophy. "In the fullness of time, the files you create are more important than the tools you use to create them."

**On AI specifically:** Rather than embedding AI into the product, Ango created obsidian-skills to teach agents Obsidian's formats. His tweet launching it: "I'm starting a set of Claude Skills for Obsidian... so far they're centered around helping Claude Code edit .md, .base, and .canvas files."

**On community vs AI:** "Obsidian prioritizes community because genuine productivity, especially in personal knowledge management, thrives on human connection, shared learning, co-creation of workflows, and collective problem-solving. While AI can augment, it cannot fully replace the organic, collaborative intelligence of a vibrant user community."

## Strategic Direction Signals

**What Obsidian IS investing in (2025-2026):**
1. **Bases** — Database views as a core plugin (competing with Notion databases)
2. **Mobile 2.0** — Native widgets, voice command integration
3. **Keychain** — Secure API key management for plugins
4. **CLI** — Official Obsidian CLI (1.12+) for automation and agent interaction
5. **Commercial license removal** — Lowering barriers to organizational adoption

**What Obsidian is NOT investing in:**
1. **Embedded AI** — No AI features in the core product
2. **Real-time collaboration** — No roadmap commitment despite 2,200+ votes
3. **Official MCP server** — Community feature request unanswered
4. **Team/enterprise features** — No admin console, SSO, compliance features
5. **Web version** — Desktop/mobile only (web access requires screen.garden)

## Adversarial Analysis: Where Obsidian Won't Go

Based on the philosophy and team composition, Obsidian is structurally unlikely to:

1. **Add LLM compute to the product.** The "file over app" philosophy values simplicity and durability. LLM features add complexity, cost, and vendor dependencies that conflict with core principles.

2. **Build real-time collaboration into core.** Local-first architecture makes this architecturally hard. The team is 18 people — building a CRDT collaboration layer is a major engineering investment they've shown no interest in making.

3. **Raise venture capital.** "100% supported by users, not investors" is a core identity statement. VC funding would require growth targets that conflict with the deliberate, community-first approach.

4. **Build an enterprise sales motion.** Making commercial use free (removing the commercial license) is the opposite of building enterprise pricing tiers. Obsidian wants organizational adoption but not enterprise sales complexity.

5. **Open source the core app.** The proprietary app is the moat that enables the free-with-paid-services model. Open sourcing would undermine Sync/Publish revenue.

## What This Means for a Competitor

Obsidian's positioning creates clearly defensible territory AND clearly open territory:

**Defensible (hard to beat Obsidian at):**
- Local-first, privacy-first single-player experience
- Plugin ecosystem breadth (2,736 plugins)
- Community depth and loyalty (60K+ active community members)
- "File over app" trust and brand equity
- CEO's personal thought leadership and credibility

**Open territory (where Obsidian won't compete):**
- Agent-native: agents as first-class participants, not filesystem visitors
- Collaborative: real-time multiplayer as a core primitive
- Git-native: version control, branching, PRs for knowledge
- Agent attribution: separate agent commits, review workflows
- Web-first: no desktop app required
- LLM-integrated: semantic search, AI-assisted organization as core features
