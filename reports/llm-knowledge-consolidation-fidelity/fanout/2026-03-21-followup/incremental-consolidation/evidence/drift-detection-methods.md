# Drift Detection Methods

## Sources
- "Incremental Learning with Concept Drift Detection and Prototype-based Embeddings." arXiv:2404.02572 (2024).
- "Unsupervised Concept Drift Detection from Deep Learning Representations in Real-time." arXiv:2406.17813 (2024).
- DriftLens: https://github.com/grecosalvatore/drift-lens

## DriftLens: Embedding-Space Drift Detection

An unsupervised statistical drift detection technique that:
- Exploits internal representations (embeddings) generated by the model
- Reduces noise and complexity vs raw input comparison
- Works for unstructured data: text, images, audio
- Uses distribution comparison in embedding space to detect when data distribution has shifted

## AdWin (ADaptive WINdowing)

One of the most popular supervised drift detection algorithms:
- Takes individual scores (model losses, p-values) as input
- Dynamically adjusts window size to estimate change points
- Detects drift without requiring a fixed window size

## Prototype-Based Drift Detection

From the 2024 paper on graph stream classification:
- Maintains representative prototypes for each class/cluster
- Detects drift when new data points deviate significantly from prototypes
- Uses incremental learning to adapt prototypes over time

## Applicability to Incremental Consolidation

For detecting when accumulated incremental updates warrant full re-consolidation:

1. **Embedding distribution shift**: Compare embedding distribution of incrementally-added claims vs original consolidated claims. Significant divergence suggests structural drift.

2. **Coverage decay**: Track what percentage of source material is represented in the consolidation. If coverage drops below threshold, trigger rebuild.

3. **Conflict accumulation**: Count the number of contradicting/invalidated claims. High contradiction rate suggests the knowledge domain has shifted.

4. **Community coherence**: Monitor intra-community similarity in graph-based systems. Declining coherence suggests community boundaries need refresh.

## Key Insight for /consolidate

Three practical drift triggers for the /consolidate skill:
1. **Conflict rate**: If >N% of new claims contradict existing claims, trigger full rebuild
2. **Coverage metric**: If consolidated output covers <X% of source claims (measured by embedding retrieval), trigger rebuild
3. **Staleness timer**: After N incremental updates without full rebuild, force one
