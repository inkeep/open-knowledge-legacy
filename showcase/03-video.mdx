# Video

Pure HTML5 `<video>` wrapper. Self-closing — no children slot. No URL sniffing; YouTube/Vimeo embeds use a raw `<iframe>` in MDX (Mintlify's pattern).

## Basic

Minimal props — `src` is the only required field. Controls default to on.

<video src="https://interactive-examples.mdn.mozilla.net/media/cc0-videos/flower.mp4" />

## With poster

A `poster` image shows before playback starts.

## With explicit dimensions

Explicit `width` and `height` keep layout stable while the player loads.

<video
  src="https://interactive-examples.mdn.mozilla.net/media/cc0-videos/flower.mp4"
  width={640}
  height={360}
/>

## Autoplay + muted + loop

`autoplay` without `muted` is blocked by every modern browser — always pair them for reliable autoplay. `loop` repeats on end. `playsinline` is required on mobile Safari for inline playback (otherwise it goes fullscreen).

<video
  src="https://interactive-examples.mdn.mozilla.net/media/cc0-videos/flower.mp4"
  autoplay
  muted
  loop
  playsinline
/>

## No controls, auto-restart

Useful for hero backgrounds — muted, looping, no chrome.

<video
  src="https://interactive-examples.mdn.mozilla.net/media/cc0-videos/flower.mp4"
  autoplay
  muted
  loop
  playsinline
  controls={false}
/>

## Preload strategy

- `preload="none"` — don't prefetch until user initiates playback
- `preload="metadata"` — prefetch dimensions + duration only (default, good tradeoff)
- `preload="auto"` — prefetch the whole video

<video
  src="https://interactive-examples.mdn.mozilla.net/media/cc0-videos/flower.mp4"
  preload="none"
/>

<video
